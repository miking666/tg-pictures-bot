# Бот-открыточник

Telegram-бот, который принимает текст или голосовое сообщение, генерирует фон открытки через Gemini и накладывает поздравление поверх изображения с помощью Pillow.

## Что умеет

- принимает текстовое поздравление;
- распознаёт голосовые сообщения через Gemini;
- предлагает набор стилей inline-кнопками;
- генерирует уникальный фон без текста с учетом темы и свободной зоны под надпись;
- программно рисует крупный кириллический текст на картинке;
- возвращает готовую открытку в Telegram.

## Стек

- Python 3.10+
- aiogram 3
- Google Gen AI SDK (`google-genai`)
- Pillow
- python-dotenv

## Структура

- `bot.py` — точка входа и Telegram-хендлеры;
- `config.py` — загрузка настроек из `.env`;
- `styles.py` — словарь стилей открыток;
- `image_gen.py` — Gemini-интеграция и наложение текста;
- `fonts/` — шрифт с поддержкой кириллицы;
- `.env.example` — шаблон переменных окружения.

## Подготовка

1. Создайте бота через `@BotFather` и получите `BOT_TOKEN`.
2. Получите `GEMINI_API_KEY` в Google AI Studio.
3. Создайте виртуальное окружение и установите зависимости:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

4. Создайте `.env`:

```bash
cp .env.example .env
```

5. Заполните в `.env` обязательные переменные:

```env
BOT_TOKEN=...
GEMINI_API_KEY=...
```

## Запуск

```bash
python bot.py
```

## Переменные окружения

- `BOT_TOKEN` — токен Telegram-бота.
- `GEMINI_API_KEY` — API-ключ Gemini.
- `GEMINI_TEXT_MODEL` — модель для генерации промпта, по умолчанию `gemini-2.5-flash`.
- `GEMINI_AUDIO_MODEL` — модель для распознавания голосовых, по умолчанию `gemini-2.5-flash`.
- `GEMINI_IMAGE_MODEL` — модель для генерации фонового изображения, по умолчанию `gemini-2.5-flash-image`.
- `DATA_DIR` — директория для runtime-данных, по умолчанию `.data/`.
- `BUDGET_FILE_PATH` — путь к файлу бюджетной статистики.
- `FSM_STORAGE_PATH` — путь к файлу FSM-состояний.
- `CARD_FONT_PATH` — путь к `.ttf` шрифту.
- `CARD_IMAGE_SIZE` — размер итоговой картинки, по умолчанию `1024`.
- `MAX_GREETING_LENGTH` — максимальная длина текста для открытки.

## Как работает бот

1. Пользователь отправляет текст или голосовое.
2. Если пришло голосовое, бот скачивает файл через Telegram API и отправляет байты в Gemini.
3. Бот сохраняет текст во временном состоянии и показывает стили кнопками.
4. После выбора стиля Gemini генерирует фон без текста и оставляет композиционное место под надпись.
5. Pillow накладывает поздравление поверх изображения локально, поэтому кириллица не зависит от качества рендера модели.
6. Бот отправляет готовую открытку.

## Замечания

- Для production лучше держать бота на сервере вне РФ, если Gemini недоступен напрямую.
- Runtime-данные и FSM теперь сохраняются в `.data/` и переживают рестарт процесса.
- Для горизонтального масштабирования следующим шагом все равно стоит вынести FSM и бюджетное состояние в внешнее хранилище.
